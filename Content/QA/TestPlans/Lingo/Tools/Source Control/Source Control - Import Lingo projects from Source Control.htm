To import a Lingo project from Source control, got to: 

  • !File > Import Project Verify that the option has a tooltip and it's correct.
  • !View > Source Control Explorer > Import Project. Verify that the option has a tooltip and it's correct.

Select Source Control Provider dialog

Controls:

  • Providers drop menu: Non editable drop down control enables you to select SC provider to import Lingo project From. Fields and description should match the selected provider:
    • TFS
    • SVN
    • Git
    • P4V
      • Microsoft Team Foundation Server: Selecting this provider should show the description and fields shown below:
        • Server: Editable drop down control where you can either enter the full server name or click the browse button to select a TFS server. Clicking browse would show the Select Team Foundation Server Project Collection dialog

          Controls:

          • Green plus button: Invokes the Add Team Foundation Server dialog
            • Name or URL of Team Foundation Server: Server machine name hosting the TFS service. - Test with invalid servers
            • Path: Default value is set to tfs

              Note: Entering invalid connection details should add the TFS server to the server list

            • Port number: Default value is set to 8080

              Note: Entering an invalid port would return this exception

            • Protocol: HTTP or HTTPS radial button control. Verify that selecting HTTPS would deselect http and reflect the update in the preview.
          • Red X button: Removes an added TFS server from the Select Team Foundation Server Project Collection dialog/ Server list section - This button should be grayed out when there's no server selected from the Server list section.
            Choosing OK should remove the server, Cancel should not do anything.
          • Team Project Collections: Lists the Team projects available on the selected TFS server. - Test that selecting different servers updates the list of available TFS team projects for that server
          • Preview: Lists the Selected TFS path including selected server and team project collection. This field should not be modifiable.
          • Cancel: Cancels out the Select TFS project collection dialog
          • OK: Inserts the selected TFS project collection into the Server field
        • Team Project: You can enter the path by typing it. Also, you can click the browse button to invoke the Browse Source Control Folders dialog - Clicking OK should populate the selected Team project in the Team project field. .
      • Subversion: Choosing subversion should show only one field to provide the server URL and the description as shown here
        • Server: Editable drop down control where you can provide the SVN repository path here. Providing an invalid path should give this error when clicking Next.
      • GIT: Choosing GIT should show the git related fields and description .
        • Remote:  Editable drop down control where you can enter the remote repository URL.

          Note:
          Test entering a blank and invalid remote URL. For remote you should get
          For invalid remote, you should get these two errors When clicking Next then browse.

        • Name: Provide user name used for importing project /file(s) from git - Test leaving this field empty and press OK. It should give this error
        • Email: Provide an email address- Leaving this field empty would return an error
        • Save per: Drop down control specifies saving per user or project.
          • User Saves the files in your local folder. Select this option if you are working with other tools (e.g., Tortoise) or if you want to use the same user identity across multiple projects.
          • Project Saves the files locally to your project. Select this option if you want to use different identities for each project.
      • Perforce: Choosing Perforce should show the Perforce related fields and description
        • Server: Editable drop down control where you can provide the perforce server URL:Port. Providing an incorrect server should provide this message after trying to browse for location
