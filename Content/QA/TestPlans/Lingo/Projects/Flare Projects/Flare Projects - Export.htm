Flare Projects - Export

Opening the UI

To Export any translated content: 

!File > Export

Tooltip should say Export to Flare - Export translated files back to Flare.

UI Test Cases:

Export Finished Translation dialog:

 

UI Controls

  • Project to Export: Displays the name and path of the current Lingo project - Test path with special characters verify path is rendered correctly.
  • Destination Filename: Lists the current Flare project name. - Test Entering a different name with spacial characters verify that the exported project is using the given name

    Note: Exporting with no FileName would return this error:

  • Destination Folder: This field should be blank by default. - Test Exporting to Mapped network drive, Network location and UNC
  • Browse: Opens the Browse For Folder dialog, which lets you select a different folder to store the project files.
  • Export: Click this button to export the project.
  • Cancel: Cancels out the export dialog - Test that it's being canceled when pressing Esc key on the keyboard.
  • Make a zip file after export: Checking this option would create a zipped copy of the exported project.

 

Overwrite existing project dialog:

If there's a project with the same name in the exporting folder the prompt below should appear. Clicking OK would overwrite the changed files, Cancel should cancel the export. Test deleted files via a diff and export to overwrite

  • Clicking “OK” or the X or Esc on the keyboard should finish the export process.

'Exporting...' progress dialog:

 

 

 

  • Open Exported Project: Opens up the exported Project/ file or folder depending on the translated content.
  • Save Log: Save the log if there were errors importing the file(s) /project to be translated - Verify that Save log button is grayed out after the log is saved.

    Log folder should be created in the project folder with the saved mclog file in it.

  • Close: Closes the Exporting dialog.
