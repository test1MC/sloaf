Flare Projects - Import

Opening the UI

See the New and Open Project Test plan for information on importing projects.

 

  • New and Open Project - UI.htm

 

Importing a flare project into Lingo adds a new dialog that comes after Select files or projects to translate dialog and the Select Translation memories to use for this project dialog. The dialog details is listed below:

Start a new Project Wizard- Select Flare project Settings

Controls:

Select variables to flatten section:

Select Variables to flatten section:

  • Flatten (left panel): Check-box column that is not sortable.
    - Selecting this check-box would select all variables within that variable set (listed to the right) to be flattened.
    - Deselecting the this check box for a variable set would deselect all variables belonging to that variable set.
  • Variable Sets: Shows the available variable sets with their folder paths. This column is sortable.

    Notes:
    - Highlighting any row on the left panel should show all variables belonging to that variable set to the panel to the right.
    - Clicking back and then Next should always highlight the 1st variable set and how its variable on the right panel.

  • Flatten (right panel): Check-box column that is not sortable.
    - Selecting to flatten variables would show a square in the parent variable set check-box if not all variables are selected.
    - Selecting to flatten all variables belonging to a variable set, should show a check mark for the variable set flatten check-box.

Selected flattened variables are going to be converted to text in the files they exist in.

  • Names: Lists the variable names contained in the selected variable set. This column is sortable.
  • Definitions:  Lists the variable definition associated with the highlighted variable. This column is sortable.
  • Comment: Lists comments of the highlighted variable. This column is sortable.
  • Convert sub-segment snippets to text: This converts the inline segments to text.
  • Create Language skin: This option creates a language skin upon lingo project creation to be translated in the target language and be in the exported translated project.
  • Back: Goes to the Add files/ Projects dialog
  • Next: Goes to the Select TM dialog
  • Finish: Finishes creating the project
  • Cancel: Closes the Start New Project Wizard with no changes.
  • Filter by target: Drop menu control that should list the imported project targets.

    - Parsed files listed for translation should be associated with the selected targets only. Other files should not be parsed for translation.
    - Exporting the project should include all files with only the parsed files translated.

  • Filter out condition: Selecting this option enables the Select Conditions... button and let's you select conditions to exclude by selecting them from the Select condition dialog

Select Conditions... :

This dialog Lists all the condition tag sets and tags in the imported flare project

  • Exclude (left side pane): Check-box control if checked all condition tags belonging that tag are checked to be excluded.  This column is not sortable.
  • Condition Tag set: Lists the condition sets included in the imported flare project. This column is sortable.
  • Exclude (right side pane): Check-box control if checked, the selected condition tag is marked to be excluded. This column is not sortable.

    Note:
    - Selecting all condition tags belonging to a condition tag set should show a check mark for the associated condition tag set where as selecting some of the condition tags would put a square in the parent condition set tag check-box.

  • Tags: This column lists the condition tags contained in the selected conditions tag set in the left panel. This column is sortable.
  • 3rd column: currently listing the parent condition tag set. Should be changed to list the comment Bug 127823.

    - After the condition tags are selected, they're added to the field below Filter out condition option.
