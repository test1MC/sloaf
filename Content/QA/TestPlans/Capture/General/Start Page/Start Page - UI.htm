Start Page - UI

Opening the Start Page

The Start Page should be open and docked in the MDI using Flare's default layout. If the Start Page is closed or not currently in focus, it can be reopened or given focus by:

!View | Start Page

Click the Open Start Page button in the Standard Toolbar

Start Page - UI Test Cases

By default, the Start Page will appear open in the MDI. It is still, essentially, a pane that can be floated and docked anywhere. Right-click the Start Page tab and select Floating to dock it somewhere else and test the following:

Open an Existing Files

 

Open Existing Files: Lists recently opened Capture Files. Clicking on any listed file will open that file in the current instance of Capture

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • Files are listed with “FileName” an extension and then “Path”
  • Only Capture files should appear in this list
  • Tooltips with full paths will display when hovering over the links
  • Clicking the link will open the file
  • The most recently used project should be listed on top
  • Files removed from !File | Recent Files | Manage Recent Files will be removed from the list
  • Pinning files should be an available option
  • Pinned files should be a the top and within that section most recently used will be listed on top

Getting Started

Getting Started: Lists ways to create or open image files. Included as part of the Start Screen.

 

UI Controls (Tooltips in parentheses):

  • Capture: Launches the Capture Region.
  • New: Opens "Create New Image' dialog for creating images from Profiles..
  • Open: Opens the Open File dialog.

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • Icons are correct
  • Tooltips with full paths will display when hovering over the links

Help Resources

UI Controls (Tooltips in parentheses):

  • Browse the Help System: Launches the Help TOC pane.
  • Browse PDF Guides: Opens "PDF_Guides.htm" from the Help system.
  • Live and Recorded Webinars: Opens dedicated page on our website.
  • Search the Help System: Opens the Help Search pane.
  • Visit the Community: Opens https://forums.madcapsoftware.com/ in your browser.
  • Visit MadBlog: Opens http://www.madcapsoftware.com/blog/ in your browser.

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • Icons/Images are correct

News and Updates

Test Cases

  • The correct pages open for all sections
  • The correct product and version should be populated in the fields for the bug reporting link (including "Beta" or "Early Experience").
  • If there is no Internet access, the News and Updates Section should display a message indicating such.
