UI

General

UI Controls

  • Minimize to System Tray: Lets user keep Capture open but hide it in the system tray.

  • Auto-Reload Documents: Select to reload documents automatically. In other words, if you have topics or editor pages open and then close the project, this option ensures that those same documents are reopened the next time you open the project.

  • Auto-Save Documents: Select to periodically auto-save any open files.

    • "#" minutes: You can specify the number of minutes that should elapse between each auto-save.

  • Close others when opening new document: An option which tells Flare to close other documents as more documents are opened based on the behavior set here:
    • No action: The default option. Opening new files will have no affect on files already open. All files will remain open.
    • Close others: This option automatically closes all other open documents. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with no changes: This option automatically closes all other open documents that do not have unsaved changes. If you have files open where changes have been made but not saved, those files will remain open until you save and close them.
    • Close others with same extension: This option automatically closes all other open documents that share the same file extension. For example, let's say you have a topic, a style sheet, and a target open. If you open another topic, the topic file that is currently open will be closed, but the style sheet and target will remain open. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with same extension and no changes: This option is the same as the previous option, except that files with unsaved changes will not automatically be closed. Those files remain open until you save and close them.
  • Reset "Show Once" Dialogs…: There are some dialogs in Flare that open just once to give you a message or warning when something changes or you switch modes. An example of this is the dialog that opens when you first switch between layout modes in the XML Editor. These dialogs have a check box labeled "Show this dialog." If you deselect that checkbox, you will not see the dialog again. However, you can use this reset feature to display that dialog (and all other "Show Once" dialogs) again the next time.
  • Automatically resize low DPI images:  Selected by default, this newer, more precise resizing of low DPI images is used. If you do not select this option, the behavior of Version 9 and earlier is used.

 

Interface tab

UI Controls

  • Ribbon: Selecting this option should enable the Ribbon-style UI interface
  • Tool Strip: Selecting this option should enable the classic (V7 and previous) UI interface
  • Theme: A drop-down menu allowing you to choose between the three available Flare UI themes
    • Black: Selecting this should enable our Black UI theme
    • Classic: Selecting this should enable our Classic UI theme
    • Silver: Selecting this should enable our Silver UI theme

Capture

UI Controls

  • Capture Sound Effect: Lets you choose whether you want to hear a recording sound when you capture an image.
  • Hide on Capture: Lets you choose whether you want the Capture application to be minimized when you capture an image.
  • Capture Mouse Cursor: Lets you choose whether the mouse cursor is included in a screen capture.
  • Automatically Send Captured Image to Clipboard: Lets you automatically send open images to the Windows clipboard.
  • Enable Time Delay: Lets you enable the time delay feature.
  • Enable Magnifier: Lets you enable or disable the magnifier that appears when you capture new images (using the "Region" method).

 

Test Case:

  • Enable "Hide on Capture" feature, verify Capture minimizes allowing you to record\capture what's behind the application.
  • Enable "Time Delay" feature, verify Capture countdown matches "Seconds to Delay" you've set.

UI Controls

Select Global Dictionary Locations

  • Use default location: Saves the dictionaries to the default location which is C:\Users\[username]\AppData\Roaming\MadCap Software\Flare\Spelling Dictionaries
  • Use custom location: Saves the dictionary to the path provided.
    • (Browse): Opens the Select Folder dialog to browse for a dictionary location.

Installed Dictionaries Grid

  • Columns
    • Language: This column lists the names of all languages with dictionaries that are installed on your computer. Dictionaries are stored in your AppData Windows folder.
    • Code: Displays the code for each language.
    • Spell: Green dots indicate which dictionaries support spell checking.
    • Hyphen: Green dots indicate which dictionaries support hyphenators. A hyphenator is used to automatically hyphenate words in an editor based on particular word patterns, as outlined in the dictionary file. (Hyphenators are important only for certain languages.)
    • Description: This column provides a short description of each dictionary
    • Import Dictionaries...: Invokes the Select dictionary files dialog which is a browse dialog that lets you find Hunspell dictionary files (*.oxt or *.zip). You can download more dictionaries from OpenOffice.org.
