Import DocToHelp - Wizard

To create a new project by importing DocToHelp files, launch the Import Project Wizard by:

  • Tool Strip interface: !File | Import Project | Doc-To-Help Project…
  • Tool Strip interface: !File | New Project > New from Import > Import a Doc-To-Help Project
  • Ribbon interface: !File | New Project | Doc-To-Help Project
  • Ribbon interface: !File | New Project > New from Import > Import a Doc-To-Help Project

Import Project Wizard - UI Test Cases

Import Project Wizard - Dialog 1: Pick project to import

Test Cases

  • Browse for projects on a network location, mapped drive, etc.
  • Should not be able to select a non *.d2h file

Import Project Wizard - Dialog 2: Enter Project Info

Test Cases

  • Save project to a network location, mapped drive, etc.

Import Project Wizard - Dialog 3: Select CSS folder(s) for Word source documents

UI Controls

  • Add folder: Produces the Add Folder dialog which lets you select a folder containing CSS files.
  • Remove: Removes the folder selected.
  • Back: Returns to the previous dialog in the Import Project Wizard.
  • Next: Proceeds to the next dialog in the Import Project Wizard.
  • Finish: Closes the Import Project Wizard, invokes the Import Progress dialog and proceeds to import the project files.
  • Cancel: Closes the Import Project Wizard and does not import anything.

Test Cases

  • All CSS files in the folder you select should be merged into one file correctly.

Import Project Wizard - Dialog 4: Select the Flare project language

Import Progress

Test Cases

  • Canceling the import process should delete all temporary files that were made during the import. You will see them in Windows Explorer in the project folder, but once you click Cancel they should be deleted.
