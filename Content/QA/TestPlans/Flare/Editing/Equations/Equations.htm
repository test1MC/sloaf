Bug Base Product Area:

  • Equations

 

Description:

Flare supports Mathematical Markup Language (MathML), which is a way to describe mathematical notations in XML. Recommended by the Word Wide Web Consortium (W3C), MathML in Flare allows you to embed virtually any kind of mathematical equation in the XML Editor.

 

Sample MathML code:

<math>

<mrow>

<apply>

<power />

<apply>

<plus />

<ci>a</ci>

<ci>b</ci>

</apply>

<cn>2</cn>

</apply>

</mrow>

</math>

 

MathML code can contain unicode characters.  The usecase for our customers is that in Flare 11 and below, the design science equation editor userd unicode characters in their source view.  So if a user wants to author their equations using a previous version of Flare, they can copy & paste the source code directly from the old equation editor into the new one.  In this sample, the unicode character is &mldr;

<math>

<mrow>

<mi>a</mi>

<mo>&mldr;</mo>

<mi>b</mi>

</mrow>

</math>

 

Flare also supports LaTeX control sequences starting in Flare 12.

 

Sample LaTeX:

\left(\frac{\sqrt x}{y^3}\right)

Note: LaTeX equations containing unicode are not supported at this time: Bug 117071:Unicode latex equations are not supported

 

Our equation editor can also handle equations copied from MS Word. See this bug for more:Bug 113937:Equations copied and pasted from MS Word do not get rendered in Flare 12 equation editor.  Here's a sample:
