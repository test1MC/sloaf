Multimedia - Output

Support in Web Outputs:

Support in HTML5 Outputs: (Tripane and Top Navigation)

  • Verify that the entire <object> tag and all of its properties are removed when conditionalized out.
  • Verify that the entire <object> tag and all of its properties are unbound when conditionalized out with the "Unbind" behavior.
  • Verify that applied styles are in outputs
    • Multimedia Types Supported (.swf, .asf, .midi, .mpeg, .wma, .asx, .mp3, .mpg, .wmv, .au, .mp4, .wav, .wmx, .avi, .mpa, .wax, .mid, .mpe, .wm, .mov, .qt)
    • Verify the filters when Importing an multimedia are correct.
    • Using the multimedia test project listed at the top to check each of these multimedia types.
    • Many of the types are not supported fully, but we need to make sure we don’t crash or error out when working with these types of multimedia.
  • Verify all target frame settings for multimedia for all web output types as well as topic previews.
  • Verify that all supported multimedia file types are supported in all outputs as well as multiple web browsers.

Support in Print Outputs:

Support in PDF Outputs:

Starting in Version 11, We now support Multimedia files in PDF output.

  • Multimedia Types Supported:
    • Movie files: *.mov; *.mp4; *.mpa, *.mpg; *.mpe; *.mpeg; *.avi; *.swf
    • Audio Files: *.mp3; *.mid; *.midi; *.wav
    • 3D files: *.u3d
    • YouTube/Vimeo- Currently, these will not play. However, you should get the following warning: " YouTube/Vimeo not supported in this output. Replaced with static images.".

 

  • Verify static image is set as placeholder for YouTube/Vimeo links.

Support in EPUB Outputs:

  • Multimedia is not supported.
  • Verify the following build warnings:
    • 130012- Plug-in reliant format 'Media-type' is unsupported by EPUB and will be stripped from the output
    • 40019- Unsupported file format. This file will be skipped in the output: filename
    • 130012- Remote videos cannot be viewed in the EPUB format, and the file will be stripped from the output (Applies to YouTube/Vimeo)

Support in DITA Output:

  • For DITA output, the <object> tag will become a <object data="Multimedia/mid.mid"></object> tag showing the relative path to the multimedia file.
  • 3D modeling not supported. Verify the following warning message:
    • 90013- MadCap:model3d found in file: "filename". It will be removed from the output.
  • Verify embedded YouTube/Vimeo video links are displayed correctly in the output.

 

Compiler errors:

  • Verify you get the message: Unsupported file format. This file will be skipped in the output. Verify it continues to build output.
