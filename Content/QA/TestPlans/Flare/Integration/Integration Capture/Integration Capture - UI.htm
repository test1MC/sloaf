Screen Capture dialog

 

 

UI Controls

  • Help Opens "Screen Capture Dialog" Dynamic Help
  • Close Closes Dialog box and does not save changes
  • File name File name for captured item. Default is Capture + incremented number
  • Folder Folder to save Capture image in. Default should be Resources/Images
  • Profile or format window Display available profiles and formats and allows for them to be selected
  • Capture UI Element buttonCreates a movable capture selection box for capturing UI Elements
  • Capture Application Opens "the "Select Window" dialog
  • Capture Region Opens a re-sizable selection box
  • Show Magnifier When Checked, should open the magnifier window when a capture is being performed
  • Launch MadCap Capture editor When Checked, should open MadCap Capture application after a capture has been completed
  • Cancel Closes the dialogue/does not perform a capture

Capture Region

 

 

  • The capture region box will have 8 resize points located around it
  • Each point can be used to resize the capture region
    • Middle boxes can be used to resize either left/right or up/down
    • Corner boxes can be used to resize the entire capture region
  • The region box will also have a cross-hair in center
    • Clicking and dragging this box will move the capture region accordingly
  • Verify resizing the capture region box will still result in an accurate capture
  • Three buttons will be displayed in the lower right corner
  • Verify capture region using all profiles and formats

 

  • Capture Button: Click to capture what lies within the boundaries of the region
  • Width and Height: Values listed change when resized using resize points. Also can manually enter values to adjust the capturing region
  • Pick a Screenshot Size (Dropdown): Brings up the options context menu
  • Close Button/Cancel: Will cancel the operation and return the user to Capture

Image Viewer

To open the Image Viewer double- click an image in the Content Explorer

 

The local toolbar will contain one button ("Edit this file with MadCap Capture") if it has not been created or saved in Capture or two buttons, with the second being "Regenerate Captured image".

  • Edit this file with MadCap Capture: Will open the file in MadCap capture.
  • Regenerate Captured image: Used to update image to its Capture state. (Test: Edit Captured image in another program such as MS Paint, save, the click this button. The image should revert back to its Capture state and remove the changes made outside Capture. Test2: Also another way to get rid of trial watermarks once Flare and/or Capture are activated)

 

Conditions and Variables

Like Mimic, conditions and variables from a Flare project linked to the image will be available for use in Capture. However, in Capture, the project is already linked to the image, so all variables and conditions will be available in Capture when editing an image that is in a Flare project in Capture.
