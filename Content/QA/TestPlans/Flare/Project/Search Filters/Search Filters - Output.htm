Support in HTML5 Output:

  • By default, no Search Filter Sets will be included in the output. In order to include a Search Filter set in the output, you must associate a Search Filter Set with the Target file in the Target Editor, in the Advanced tab. Only one Search Filter Set can be used per target:

 

UI Controls

  • Filter File: Drop-down displays a list of all Search Filter Set files that exist in the project. You can select one to associate with the Target.
  • Edit: Opens the selected Search Filter Set file in the Search Filter Set Editor.

    HTML5 Tripane

    HTML5 Top Navigation

Test cases

  • Ordered search filters should display the order specified in the search filter file.  If no order is set, list will be alphabetical.
    • Note: Ordered search filters are not currently supported in HTML5+ output, they will display in alphabetical order (bug 93786)
    • Note: If 'Include server-side' in HTML5 target | Publishing tab is selected, no search filters will show up if you view the output before doing the steps to publish to IIS.
    • Note: Search filters are not supported in merged HTML5/HTML5+ projects using TopNav or No Skin.
  • Test that applying the filter limits the search results to only those topics which contain the Concept link.
  • Test that single words, multiple words, numbers, words with international characters, etc. set in the Search Filter Set Editor generate properly to the output
  • Drop-down generates a scroll bar if the filter list is long.
  • Up Arrow and Down Arrow keys navigate through Filter list. With each new filter selection, the results list should refresh.
  • Character keys select and scroll through filters beginning with that character. (Ex. If you have three filters called Cars, Cats and Cows, keying “C” on the keyboard will cycle through those three in the order in which they appear in the filter list)

Support in Web Help Outputs:

Skin Editor setup:

  • Since the Search Filter UI is part of the Search Pane in the output, make sure the Search is toggled "on" in the Basic tab of the Skin Editor. By default, it will be toggled "on."
