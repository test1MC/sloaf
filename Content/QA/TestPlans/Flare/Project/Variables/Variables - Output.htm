Output: Overriding Variable Definitions in Targets

The variables that you create and define in the Variable Set Editor are available to your entire project. However, if you want the definition for a variable to be different in a particular target, you can override the project-level definition for that target in the Target Editor.

 

How to override a variable definition in a target:

 

  1. Open any target | Variables tab
  2. Select a variable set
  3. Click in the Definition cell for the variable that you want to override and press F2 on your keyboard.
  4. Enter a different definition and press Enter on your keyboard.

    Test replacing the definition with a space (essentially removing the definition) - See Bug 99993

  5. Press CTRL+S or click to save your work.

 

The new variable definition will be used in the output for this target. However, for the other targets in your project, the original variable definition will be used.

Note: If you preview a topic, the variable definition associated with the primary target is displayed.

Note: If you click the Edit Set button at the bottom of the tab, the Variable Set Editor opens. When you use that editor, you are editing variables for all targets, not those for a specific target.
