Search UI and Options- HTML Help - CHM

Enabling Search and how it works in the output

  1. Enable the search feature in the skin by using the skin editor

  2. Basic search in the output

  3. Advanced Search options are enabled in the Standard Skin Editor for CHM output

  4. Advanced Search enabled in the output

  5. Associate the skin with search enabled with the target that is going to be generated.

  6. Search in the output includes Title, Location, Rank, and results are highlighted

    If searching any term in the CHM output gives a “No Topics Found” message, then the itcc.dll needs to be registered

    1. In Windows Explorer, open the computer's System32 directory. The path is C:\Windows\System32.

    2. Right-click the application file named cmd.exe and select Run as administrator from the context menu.

    3. Type cd to change directories and press the SPACEBAR. Then type the path to the Flare \bin directory such as below (replace the version number accordingly)

      cd C:\Program Files (x86)\MadCap Software\MadCap Flare V11\Flare.app\Resources\Bin

    4. Press Enter.

    5. At the command prompt, type: registeritcc.bat. Then press Enter. This runs the batch file. If the registration is successful, a message tells you so.

    6. Click OK to acknowledge the message.
    7. Close the command prompt.

    After registering the DLL, build your Microsoft HTML Help output again. Then retry your search in the output.

Option to make a topic not searchable in the Topic Properties is not supported for CHM output

Search Filter Sets are not supported for CHM output

Search terms are highlighted by default for CHM output and the styling cannot be changed

There is no option to include / exclude stop words - they are included by default

Keywords / Index terms are always excluded from search
