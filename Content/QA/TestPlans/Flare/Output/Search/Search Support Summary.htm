Operators Description Usage DotNet CHM WebHelp HTML5 Air Mobile WH Plus HTML5 Plus
AND Use to narrow the search and retrieve only the topics that contain all of the words it separates. Type AND between two or more terms. This is not case sensitive. If you use the word "And", It will only give back results that have the word "and" in the topic.  It does not work in boolean logic
Instead of the word AND, type the plus symbol (+). This works because the default logic is AND - the logic character is not used
Instead of the word AND, type the ampersand symbol (&). This works because the default logic is AND - the logic character is not used If you use the & operator it will give all the results which have the first term and disregard the second term in the search
OR Use to search for two or more words. This broadens the search results by retrieving topics that contain any of the words it separates. Type or between two or more terms. This operator is not case sensitive. When separating terms with a blank space, the OR is always inferred.
Type the pipe symbol (|) instead of typing OR.
NEAR Select from the operator list to find topics that contain matches that are in close proximity to each other. This is only available in Microsoft HTML Help. The operator must be selected from a list.
NOT Use to search only for matching topics that do not contain the term following the operator. Type NOT before a term. This operator is not case-sensitive.

Only works if there is a word before and after the word such as Apple NOT Fruit (same in v10 and v11). Does not work with just "NOT Fruit"
Type the exclamation mark (!) instead of the word NOT.
Use to find topics that contain one term, but not the other. Type a term, followed by the carat symbol (^), and then type another term. - logically this converts to (AND NOT)
Parenthesis Use parentheses to combine boolean phrases and search terms. Type parentheses ( ) around the combined terms.
