Source Control - Pending Changes

Opening the Pending Changes

The Pending Changes should be open and docked along the left side using Flare's default layout when opened by:

  • !Source Control| Pending Changes
  • Clicking the Source Control button on the standard toolbar and selecting the Pending Changes menu item.
  • !File | Source Control | Pending Changes
  • Right-click a file bound to Source Control and selecting the Source Control | Pending Changes context menu item.
  • !View|Source Control Explorer- Select Pending changes from drop down or Window (See Source Control - SC Explorer.htm)

Pending Changes - UI Test Cases

Default view docked in main dock area:

UI Controls

  • Show/Hide Columns: Brings up a columns window that lets you select which columns to show. This window also lets you apply a color to the column and move the column up or down.
  • Open: Opens the selected file.
  • Properties: Opens the properties of the selected file.
  • Refresh: Refreshes the Pending Changes.
  • Check file(s) into Source Control: Check the selected file into Source Control. Will display 'Commit' for SVN and Git or 'Submit' for Perforce.
  • Show Check Outs from Other Users: When Toggled on, show files that are checked out by other users (Not applicable to Git).
  • Show Out of Date Items: When Toggled on, show files that are out of date compared to the server version in Source Control (Not applicable to Git).
  • Show Deleted Items: When Toggled on, show files that have been deleted from Source Control.
  • Show Locked Items: When Toggled on, show files that are locked by users (Not applicable to Git).
  • Filter: Lets you filter the visible files in the Pending Changes to a particular file type. Verify all files listed are correct and that any new Content file types are added.

Columns

  • Mapped Resource: Shows an icon if the file is mapped to an external resource such as SharePoint.
  • File Icon: Shows the icon of the file, the same one as the one used in the Content Explorer for that file.
  • Conditional Indicator: Shows the current condition of the file, whether pending add/delete, out of date, checked out, etc.
  • Name: Shows the name of the file.
  • Status: Shows the status of the file. Statuses that could be shown include: Checked Out (By Other User),Checked Out, Locked (By Other User), Out of Date, Deleted, Pending Add, Pending Delete, Renamed/ Moved
  • User: Shows the user of the currently selected file, most important for determining the user that has a file checked out (Not applicable to Git).
  • Path: Shows the relative path of the file.

Right-Click Menu

Content file right-click menu:

 

  • Menu items are correct
  • Spelling and Grammar are correct
  • No duplicate accelerators
  • All accelerators function
  • Keyboard can navigate and select options
  • Menu is easy to use and understand

 

Right-click menu items:

  • Preview: Opens a Preview of the selected content file (Only active for topic files).
  • Open: Opens the selected Content file in the MDI.
  • Open with: Gives options to open the Content file in the Internal Text Editor, Internal Browser or another application.
  • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan).
  • Delete: Removes the selected Content file from the project.
  • Rename: Allows you to rename the Content file
  • Convert to XML It converts file(s) to XML
  • Properties: Opens the Properties dialog window for the selected file.
  • View Links: Opens this file in the Link Viewer.
  • Locate in TOC: Opens the TOC Editor and locates the content file in the TOC. If the content file occurs in multiple TOCs, this will open the
  • Locate in Explorer: Opens the Content Explorer and locates the content file in the Content Explorer.

Keyboard Navigation

  • Up Arrow and Down Arrow move the selection to the next displayed node.
  • Page Up and Page Down will move the selection within the Explorer a page at a time up or down.
  • Home or End will move the selection to the very first node or the very last displayed node.
  • Insert does nothing.
  • Delete removes the selected file (Only applies to items not pending delete)
  • Keying the first character of a file will move the selection to that file.
