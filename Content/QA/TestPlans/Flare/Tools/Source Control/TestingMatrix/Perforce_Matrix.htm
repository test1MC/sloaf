Basic Testing of Perforce using Third Party Plug-in instead of Native Integration

Steps to bind a project to Perforce using Perforce Plug in

  1. Install the perforce plug-in and P4V (\\fs1\software\Perforce)
  2. Open any Flare project (not bound to source control) and go to !File | Project | Project Properties | Source Control | Bind Project
  3. Select "Third Party Plug-in", then select Perforce SCM, and click on the browse (...) button
  4. Enter in the Perforce server and user name (cent64perforce:1666, test1)

  5. Click on "New" for the workspace and create a workspace name- enter credentials as needed

  6. In the New Workspace Dialog, enter in the path to the project in the "Root" field and edit the View field to include the Project name (see screenshot - highlight parts are the modified sections - delete depot paths that are not needed)

  7. Click on "Save" to save changes and return to "Open Connection" dialog, click OK, then OK on Bind Project dialog

  8. Submit Changelist dialog appears from Perforce- click on Submit

  9. Click OK again on Open Connection dialog - Click Submit on Submit ChangeList that appears again - this time for the flprj file

  10. Click OK to close out of the Project Properties dialog
