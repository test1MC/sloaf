Knowledge: Centralized team’s knowledge to be leveraged by users

  • Create different articles edit them create different versions archive and publish.

  • Attach articles to open or closed cases

  • Related list: shows the articles related to a certain case including a version number and if a version number becomes outdated because it’s been used, it’ll be indicated using this list.

Admin setup:

  • Different Article types: about the look and feel. Define the structure of how the article will look. (FAQ article, How To article)

  • Different Data categories: when articles are created we can put them in different data categories (more than one can be used). Used to classify and search for articles. Category groups > categories> subcategories within them. Also, with roles, certain users can be given rights to view certain categories.

Steps to create a knowledge:

  • Enable the knowledge option for user from:

  • Setup (top section)

  • Administer

  • Manage Users

  • Users

  • Edit User

  • Enable (Knowledge user) option.

  • Create article type: Build

  • Knowledge

  • Article Types

  • New article Type

  • Add in a new article field:

  • From the Knowledge Article Type, click the article type label

  • Click New in the Fields section

  • Text Area (rich) this type Allows users to enter formatted text, add images and links. Up to 131,072 characters on separate lines

  • Enter Field Label and Name (similar) >Next>Next >Save

  • To enable the created Rich text field, add a page layout section:

  • Create a new page layout and see that the created rich text field is listed under the information section

  • Drag the section icon below the information section and call it my section

  • Drag the rich text field from the information section to “my section” > Save

  • Knowledge settings:

  • Setup > Customize> Knowledge > Settings > Edit

  • Allow users to create and edit articles from the articles tab

  • Allow users to add external multimedia content to HTML in the standard editor

Channels: Channels are what is displayed to an audience. Articles go into these channels.

  • Internal App (default)

  • Partner

  • Customer

  • Public Knowledge Base


Articles in Salesforce Knowledge require a template for each channel.

  • Tab template - displays each section as a tab (like browser tabs). Within each tab, fields are listed one after another. Tab is used by default when creating Articles.

  • Table of Contents template - displays sections on one page with  hyperlinks to each section (like TOC as we know it).

  • Custom template - These use selected Visualforce pages to display sections.


Steps to set templates for each channel: Go to Knowledge Article Types > (select your article – to bring up details) > Channel Displays Edit section > select template per channel.



Categories: Salesforce Knowledge uses data categories to classify articles and make them easier to find (e.g. to classify articles by sales regions and products, create two category groups: Sales Regions and Products). These are user created.

Profile permissions needed for user to be able to publish to Salesforce.

Users are assigned to a profile on SF, which is like a user group in windows. You can set different access rights on that profile which will limit the permissions of users belonging to it. Besides the Knowledge user check-box we should have for knowledge users, theses are the profile permissions needed to be able to publish to SF:

  • Under Administrative permissions:

    • API Enabled

    • View Data Categories

    • Modify All Data: needed for the ability of browsing data categories only from the destination file editor.

  • Under General User permissions:

    • Manage Articles

    • Allow view knowledge

  • Under Article Type Permissions: every article type will be listed with its own access rights

    • Read

    • Create

    • Edit

    • Delete (Needed when remove stale files and/or publish articles options are checked)
