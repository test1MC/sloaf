IgnoredWords Editor - UI Test Cases

Button (Tooltips in Parentheses):

  • (New item): Creates a new empty row
  • (Cut): Cuts the selected item(s) to the clipboard.
  • (Copy): Copies the selected item(s) to the clipboard.
  • (Paste): Pastes the item(s) from the clipboard.
  • (Delete): Deletes the selected item(s)

IgnoredWords Editor grid: Displays a list of words added to the IgnoredWords Editor.

  • Word: Displays each word that has been added to the IgnoredWords Editor.
  • Comment: Lets the user add a comment for the corresponding word. This comment is strictly for use within the IgnoredWords Editor.

Right-click menu (Tooltips in Parentheses):

  • Menu items are correct
  • Spelling and Grammar are correct
  • No duplicate accelerators
  • All accelerators function
  • Keyboard can navigate and select options
  • Menu is easy to use and understand

IgnoredWords - General Editing Test Cases

Drag and Drop

There is currently no drag/drop editing functionality within the IgnoredWords Editor

Deleting

Item:

  • Select the IgnoredWords row and press Delete on the keyboard
  • Select the IgnoredWords row and click in the local toolbar.
  • Select the IgnoredWords row and click in the Standard toolbar.
  • Select the IgnoredWords row and select Delete from the Edit menu.

Tooltips

If any Word is so long that it does not fit within the column, hovering over the cell will display the entire contents of that cell.

Conditional Tags

Conditional Tags cannot be applied to individual IgnoredWords. Conditional Tags can be applied to the actual IgnoredWords file (*.mcign), however, as of Flare V5, this tag can only be used for the purposes of Advanced Global Project importing.

Code

IgnoredWords File

<?xml version="1.0" encoding="utf-8"?>

<IgnoredWordsDictionary>

<Word>tursday</Word>

</IgnoredWordsDictionary>
