UI Controls

Spell Check:

  • (first drop down):
    • (current document): Flare checks the spelling in the active file only.
    • (documents in the same folder): Flare checks the spelling in all files located in the same folder as the current one.
    • (content folder): Flare checks the spelling for all files located in the Content Explorer.
    • (pick a folder): A dialog opens that lets you choose any folder in your project. Flare will then spell check the files within that folder.
    • (whole project): Flare checks the spelling in every file in the project.

  • (second drop down):

    • (field): You can type the file type extension in the field directly. Ex: if you want to look in all target files in the project, you can type *.fltar in the field.

    • All Files: Select this option to spell check all file types, including (but not limited to) those listed in this drop-down field.
    • Topics: Select this option to spell check only topic files.
    • Snippets: Select this option to spell check only snippet files.
    • Table of Contents: Select this option to spell check only TOC files.
    • Glossaries: Select this option to spell check only glossary files.
    • Index Keywords: Select this option to spell check only index keywords.
  • Start/Cancel: Spell check will begin with the selections made above. If the process is in progress, the label changes to "Cancel," which you can click to stop the process. All occurrences of questionable spellings are displayed in a grid at the bottom of the window pane.

  • View in File: Opens the file where the current questionable spelling occurs, highlighting it.

Not In Dictionary:

  • Ignore: Flare ignores the spelling of the word for the current session.
  • Ignore All: Flare ignores the spelling of the word and adds the spelling of this word to the Ignored Words Editor
  • Add To Dictionary: Flare adds the word to the dictionary so that it is not flagged as a questionable spelling in the future. The word will also be displayed as a suggestion for future questionable spellings that are very similar. The kind of dictionary (Global or Project) to which it is added is determined by the selection in the next field.
  • Global/Project: If you select Global, spellings of words are added to a dictionary that can be used by any projects pointing to it. If you select Project, spellings of words are added to a dictionary that can be used only by the current project.

Suggestions:

  • (field): Type your own suggestion the field or choose from the list.

  • Change: Flare changes only the current instance of the misspelled word with the alternative word selected in the Suggestions field.
  • Change All: Flare changes all instances of the misspelled word with the alternative word selected in the Suggestions field.
  • Options: This opens the Spelling tab of the Options dialog. See Spelling tab

Spell Check Grid:

  • Filename: Displays the filename with extension
  • Spelling Error: Displays the incorrect word
  • Location: Displays the location of the file (Ex: Content, Project/TOCs, etc)
  • Error Type: Displays the type of error: Typo, Repeated Word, Missing Trailing Space, Missing Leading Space, Changed. The missing space error will occur if you don't have a space before/after a variable. After changing a word by selecting or tying a suggestion, the error type will read "Changed".

Setting the Language

Setting the language of your project

Set the project language in the Start New Project Wizard when you create the project, or change it by viewing Project Properties (Language tab). Flare will use this language when spell checking your project. Test that Spell Check works correctly with different project languages set.

Setting the language of a topic

If you want a topic to use a different language for Spell Checking than the rest of the project, you can set a different language at the topic level. This can be set in the Topic Properties dialog (Language tab). It can also be set by right-clicking the html tag in the XML Editor and selecting Language from the context menu. Setting the topic language via the html tag will open the Select Language dialog.

Flare will use the project language for most of the project, but use the language you set in the Topic Properties dialog when spell checking such a topic. Test that spell check works correctly for a topic which uses a different language than the project language.

Setting the language of your content

If you have a particular paragraph or phrase in a topic that should be spell checked using a different language dictionary, you can tag any element (including a span) with a language tag. The language for a particular element is set in the Select Language dialog.

Select Language Dialog - UI Test Cases

To open the dialog:

  • Right-click an elements block bar and select Language from the context menu
  • Select an element and go !Format | Language
  • Select the specific text within an element and go !Format | Language (this will result in a span tag around the selected text)

UI Controls

  • Hide unsupported languages: Reduces the list of visible languages to those which have spell check support.
