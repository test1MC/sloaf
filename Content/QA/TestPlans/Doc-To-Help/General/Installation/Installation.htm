Installation

Testing Notes:

Since side-by-side installation is not allowed, when you install a new version (major or minor build), the previous version should uninstall automatically during the wizard.

 

New to V5 these files are created on install (instead of after launching the application):

  •  Template files
    • C:\Users\ebrown\AppData\Roaming\Microsoft\Templates
    • C:\Users\ebrown\AppData\Roaming\Microsoft\Word\STARTUP
  • My Doc-To-Help Projects folder, C:\Users\ebrown\Documents\My Doc-To-Help Projects
    • Contains a Samples folder with sample projects
      • These also live here: C:\Program Files (x86)\MadCap Software\DocToHelp\Samples so you can copy these over again if you want fresh ones
    • Contains a Doc-To-Help\Themes folder with your output themes.

Notes on Installation

  • Close Microsoft Word before beginning Doc-To-Help installation. Microsoft Outlook should also be closed if Microsoft Word is your default email editor.
  • In earlier versions of D2H documentation, there was this note about anti-virus software.  I don't think we have this issue any more and you should be able to install even with you anti-virus software running.  For reference, this is the outdated documentation:
    •  Turn off your anti-virus software. (You can turn it back on after installation is complete, although you may need to modify it later.

      • Note that some anti-virus software can limit the functionality of a range of software applications, including Doc-To-Help.

        For example, anti-virus software may:

        -Interfere with the integration between Doc-To-Help and Microsoft® Word.

        -Limit or prohibit the use of scripting technology.

        If your anti-virus software alerts you with a warning while using Doc-To-Help, it may be necessary to modify the

        properties of your anti-virus software.

  • If you have projects, templates, and style sheets from existing Doc-To-Help projects, it is recommended that you back up those files before installing Doc-To-Help.
  • To install Doc-To-Help, you must have administrative privileges.
  • Doc-To-Help requires the Microsoft .NET Framework 4.0, and will install it for you automatically if it is not found during installation. If Doc-To-Help installs the .NET Framework, you may be required to restart your machine before continuing the Doc-To-Help install. After the restart, you must log in as the same user who began the install, or the installation won't continue.
  • Silent installs of Doc-To-Help are supported.

    • To install DocToHelp silently into a specific folder with log, use the following command:

      "pathtexeifile" /qn /l*v "pathtologfile" INSTALLLOCATION="pathtofolder" /i

      Note: The /i switch must be the last switch in the command line.

      Note: To install Doc-To-Help silently, Microsoft .NET Framework 4.0 must be already installed on those machines (because the automatic .NET Framework installation requires a restart).

For reference, this is the previous documentation for using live update:

Using Live Update

Live Update is a utility that automatically checks for software updates every time you open Doc-To-Help.

If a new version is available, the Doc-To-Help Live Update dialog box will open. The update number and new feature

list will be displayed. Click the Download button to download the update.

To disable Live Update

1. Choose the File tab > Tools > Options. The Options dialog box will open.

2. Click the Updates button.

3. Select the Do not check for updates automatically radio button.

4. Click OK.

To check for updates manually

Choose the File tab > Tools > Check for Updates.

Test various OSs and Word Versions

Verify the following configurations:

  • Install on a system with German Office (\\fs1\Virtual Machines\Windows 10\x64\Germany\Windows 10 x64 Enterprise Germany + German Office 2010 x64) . On the German system, please create a project which uses the source template called Normal A4 Template, and ensure the "special formatting" ribbon displays.

  • Install on a system with Japanese Office -- copy from \\fs1\Virtual Machines\Windows 8\Windows 8.1\Windows 8.1\Japanese\x64\Japanese Office 2016 -- On the Japanese system, please create a project which uses the source template called Normal A4 Template, and ensure the "special formatting" ribbon displays.

  • Install on a German OS
  • Win 7
  • Win 8.1
  • Win 10
  • Office 2010
  • Office 2013
  • Office 2016

  • Install on a clean system (no other MadCap product has been installed on this system)

  • Install on a machine without Word. Should be prompted "Word 2010 or newer is required to run Doc-To-Help" (Regression of 97703)

 

Ideas for sanity testing on these various systems

  • Create a new project, make edits, build targets

  • Open a project created in a previous version, should prompt to upgrade and continue working. Make edits, build targets

  • Open and make an edit to a source template. Ensure that edit is reflected in any new document you create based on that template. (http://help.madcapsoftware.com/d2h3/Content/z_DuplicateTopics/D2H/About_Templates_and_Styles_D2H.htm )

  • Open and make and edit to a target template. Ensure that edit is reflected in the manual output which uses that target template.
