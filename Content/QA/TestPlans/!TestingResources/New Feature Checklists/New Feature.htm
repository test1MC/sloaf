New Feature Checklist

  • Smoke test
    • What’s it supposed to do?
    • Does it do it?
    • Does it answer the customer(s) request?
  • 3rd party requirements needed?
  • Work on both 32 bit/64 bit
  • Work on a full clean machine
    • This means the machine never had a MadCap product or other software like Office
  • Test on clean Project first
    • This eliminates chance of unknown variables affecting the feature
  • Can you test it outside of Flare? (Source control, etc…)
    • If you can, how does the outside functionality compare to our implementation in Flare?
  • DPI testing
    • Is the feature negatively impacted on high DPI monitors
  • 32 bit/64 bit testing
    • Doubly important If there are 3rd party requirements
  • Compiling
    • Impact on Build Log/Warnings/Errors
    • Impact on Build performance
  • Styles
    • Were new styles created for this feature?
    • What existing styles were impacted?
    • Do the styles for this feature work in web/print mode of XML editor?
    • Are the styles supported by major browsers?
  • Template
    • Can this new feature be used for a template?
  • Output Impact
    • Web support – All browsers? Mobile?
    • Print support
    • Other outputs
  • Usability questions
    • How many clicks? Can there be less?
    • Verify the UI make sense
    • Tab Order
  • Impact to Content Central
  • Impact on Flare as a whole?
    • Look through insert menu – quick
    • Look through module list – long
  • Big ones:
    • Global project linking
    • Sub projects/run time merging
    • Source Control
    • Integration with capture/mimic/lingo ?
  • Large project impact (Pauls, QATestplans, etc…)
  • Performance impacted? (use task manager)
    • Build/Compile time
    • Idle usage
  • Does it work on German?
    • Comma vs period
  • Do we need to increment window layout for this feature? Is there a new window pane?
