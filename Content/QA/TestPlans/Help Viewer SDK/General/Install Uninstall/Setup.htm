MadCap Help Viewer SDK

Help Viewer SDK Setup

  • MadCap Help Viewer must be installed on the machine (http://www.madcapsoftware.com/downloads/redistributables.aspx).
  • Microsoft Visual Studio 2010 must be installed on the machine as this is needed to compile the Help Viewer. Then the Flare output will be used with the Help Viewer to view the generated help system.

 

  1. Download the Help Viewer SDK from the madcapsoftware.com redistributables page (http://www.madcapsoftware.com/downloads/redistributables.aspx).
  2. Unzip the package > go to the Bin folder > verify MadCap.HelpViewerClient.dll matches the version of the API (e.g. MadCap Help Viewer Api 6.3)
  3. Package should contain 4 API's

    • HelpViewerClientTester

    • HelpViewerEmbeddedClientTester

    • HelpViewerEmbeddedClientTesterWpf

    • HelpViewerMergeModuleTester

  4. Double-click the *.sln file to open each API in Microsoft Visual Studio 2010.
  5. In VIsual Studio > Build > Build Solution
  6. In Visual Studio > Project > Open Folder in Windows Explorer
  7. The bin folder should contain either a Debug and/or Release folder. These are the generated help viewer applications that can be installed on your machine.
  8. Copy either folder to a clean VM and run the *.exe
  9. Browse for the DotNet help system created in Flare.
  10. Use the various functions in the tester against Flare generated DotNet Help.
